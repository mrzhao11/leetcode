递归函数的设计步骤

1. 函数契约（What / 输入输出）
   输入参数：描述“当前要解决的子问题”。
   返回值：这份输入对应的精确答案。
   你要在脑子里（或注释里）说清楚：
   “给我这些参数，我保证返回什么。”
2. 基本情况 Base Case（终点）
   找到问题的最小版本：已经简单到不用再递归。
   直接算完并 return。
   决定了递归一定会停下，不会无限套娃。
3. 递归步骤 Recursive Step（拆小）
   想：如何把大问题缩小一点点，变成“同类但更小”的子问题。
   相信自己：假设这个函数对“更小的问题”已经能算对（递归假设）。
   调用自己：subAns = f(更小的参数)。
4. 合并子解（组答案）
   拿到各个子问题的返回值。
   根据当前层的需求，把它们加工 / 合并，做成当前问题的答案。
   return 当前答案，履行第 1 步的契约。

`func(params):
if 基本情况:
直接返回答案  // 不递归

不是基本情况 → 递归

子解1 = func(更小问题1)
子解2 = func(更小问题2) ...

用子解1, 子解2, ... + 当前信息 → 构造当前答案
return 当前答案

`
